## 1. Abstract
This project simulates the concept of Virtual Memory management in Operating Systems. It implements two page replacement algorithms—FIFO and LRU—to handle page faults when memory is full. Additionally, it visualizes the performance difference between these algorithms using a bar graph and demonstrates Logical-to-Physical Address Translation, a core concept in paging.

## 2. Introduction
Virtual Memory allows the execution of processes that are not completely in memory. It is implemented using demand paging. When a process requests a page not in memory, a page fault occurs. The OS must then bring the page from the backing store into memory. If memory is full, a page replacement algorithm selects which victim page to remove.

The objectives of this project are:
- To simulate FIFO and LRU page replacement algorithms.
- To compare their efficiency based on the number of page faults.
- To demonstrate how logical addresses are translated to physical addresses using page tables.

## 3. System Analysis

### 3.1 Algorithms Used

#### FIFO (First-In-First-Out)
- **Logic:** The oldest page in memory is replaced first.
- **Queue Implementation:** Pages are maintained in a queue; new pages are added to the rear, and replacements happen at the front.
- **Pros:** Simple to implement.
- **Cons:** Suffers from Belady's Anomaly (more frames can lead to more faults).

#### LRU (Least Recently Used)
- **Logic:** The page that has not been used for the longest time is replaced.
- **Implementation:** Uses a time/counter or stack to track usage history.
- **Pros:** Efficient, often closest to optimal; does not suffer from Belady's Anomaly.
- **Cons:** Complex to implement (hardware support needed for overhead).

### 3.2 Address Translation
- **Logical Address:** Generated by CPU (Page Number, Offset).
- **Physical Address:** Actual address in RAM (Frame Number, Offset).
- **Page Table:** Maps Page Number → Frame Number.
- **Formula:** `Physical Address = (Frame Number * Page Size) + Offset`

## 5. Sample Usage & Output

### Input
- **Reference String:** `7 0 1 2 0 3 0 4`
- **Frames:** `3`

### Output Console (FIFO)
```text
Page 7 → FAULT | Memory: [7]
Page 0 → FAULT | Memory: [7, 0]
Page 1 → FAULT | Memory: [7, 0, 1]
Page 2 → FAULT | Memory: [0, 1, 2]
Page 0 → HIT   | Memory: [0, 1, 2]
...
Total FIFO Page Faults: 7
```

### Output Console (LRU)
```text
Page 7 → FAULT | Memory: [7]
Page 0 → FAULT | Memory: [7, 0]
Page 1 → FAULT | Memory: [7, 0, 1]
Page 2 → FAULT | Memory: [0, 1, 2]
Page 0 → HIT   | Memory: [0, 1, 2]
...
Total LRU Page Faults: 6
```

### Inference
LRU (6 faults) performed better than FIFO (7 faults) for this specific input string due to its ability to retain frequently accessed pages (locality of reference).

## 6. Conclusion
This project successfully demonstrates the working of paging and page replacement algorithms. Through simulation results, we observed that LRU generally outperforms FIFO by taking advantage of temporal locality. The project also clarified how the MMU (Memory Management Unit) handles logical-to-physical address translation.
