# Viva Questions & Answers: Paging & Virtual Memory

## Core Concepts

**Q1: What is Virtual Memory?**
**A:** Virtual memory is a technique that allows the execution of processes that are not completely in memory. It gives the illusion of a larger memory than physically available by using "pages" and "frames".

**Q2: What is Paging?**
**A:** Paging is a memory management scheme that eliminates the need for contiguous physical memory allocation. It breaks physical memory into fixed-sized blocks called **frames** and logical memory into blocks of the same size called **pages**.

**Q3: What causes a Page Fault?**
**A:** A page fault occurs when a program tries to access a page that is mapped in the logical address space but is not currently present in the main physical memory (RAM).

**Q4: What is Demand Paging?**
**A:** Demand paging is a system where pages are loaded into memory only when they are actually demanded (requested) by the executing program, rather than loading the entire program at once.

## Algorithms

**Q5: Why does LRU typically perform better than FIFO?**
**A:** LRU (Least Recently Used) considers the past behavior of the program (locality of reference). It assumes that pages used recently will likely be used again soon. FIFO replaces the oldest page regardless of how useful it might be.

**Q6: What is Belady's Anomaly?**
**A:** Belady's Anomaly is a phenomenon where increasing the number of page frames results in *more* page faults. This can happen in FIFO, but **never** in LRU or Optimal algorithms.

**Q7: Explain the logic of your LRU implementation.**
**A:** My LRU implementation uses a dictionary (hash map) to store the last access time of each page. When we need to replace a page, we find the page with the minimum timestamp (oldest access) and remove it.

**Q8: Why did you choose Python for this simulation?**
**A:** Python offers excellent list manipulation features and libraries like `matplotlib`, which made it easy to simulate the queue/stack behavior of FIFO/LRU and visualize the results quickly.

## Translation & Addresses

**Q9: How is a Logical Address translated to a Physical Address?**
**A:** The logical address is divided into `Page Number` (p) and `Offset` (d). The Page Table maps `p` to a `Frame Number` (f). The physical address is calculated as: `(Frame Number * Page Size) + Offset`.

**Q10: What is a Page Table?**
**A:** A page table is a data structure used by the virtual memory system to store the mapping between virtual addresses (pages) and physical addresses (frames).

**Q11: What information is typically stored in a Page Table Entry?**
**A:** It stores the Frame Number, a Valid/Invalid bit (to check if in memory), a Dirty bit (modified), and protection bits (read/write).

**Q13: In your code, how do you handle a "HIT"?**
**A:** A "HIT" occurs when the requested page is already in the `memory` list. In FIFO, we just print "HIT". In LRU, we update the timestamp of that page to show it was recently used.

**Q14: What happens if the page table is full?**
**A:** If the page table/memory frames are full and a new page is needed, a replacement algorithm (FIFO or LRU) is triggered to evict a victim page to make space.
